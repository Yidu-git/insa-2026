## File permissions
To list out all directories and file permissions we run the `cd` command followed by the the directory, example, to go to the `tmp` directory in the home directory we run:
```bash
cd /tmp
```
To list out the directories with permissions we run:
```shell
ls -l
```

```shell
ls -laR
```



- **`/boot`** Contains binary of OS
	- grub
- **`/dev`** Contains devices and drivers
- **`/etc`** Configuration files
	- Scripts  used in startup and shutdown
- **`/home`**
- **`/lib`** Library files either
- **`/media`** Temporary mount directory for removable devices
	- `/media/cdrom` for CD-ROM
	- `/media/floppy` for floppy drives
- **`/mnt`** Temporary mount directory where system admins can mount filesystems
- **`/opt`** optional application software packages
- **`/dbin`** Contains system binary files
	- Linux commands are located here and are used by system admins for maintenance
	- example: iptables, reboot
- **`/tmp`** Contains temporary files
- **`/usr`** Secondary hierarchy for read-only user data

## Text editor for kali Linux
Command line editors such as nano, vim, emacs exist for editing code and text in the terminal. For graphical interface options we use applications like VS Code and Sublime (Vim is also available with a graphic interface)
### Nano
Nano is the default text editor for Linux. It uses a CLI and uses shortcuts for simple operations like `^O` for save and `^X` for exit.
### Vim
Is also the default for most Linux distros and is mostly used for debugging. 
- `i` insert mode
- `:w` save
- `:q` quit
- `:wq!` force quit
- `:u` undo
- `:%!command` execute command
